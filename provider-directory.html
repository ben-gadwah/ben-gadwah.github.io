Provider Directory

(InterSystems)

Provider Directory is an application that allows hospitals to find providers, where they're working, and which insurances they take. The app had a data engine that collect sources and create a "master" record. Someone would be able to create definitions on which information from each source survived in the record and for how long. Users would also be able to make edits that overrode contributing sources. One of the main personas for this project is an Informaticist - someone who would use the Provider Directory to make sure all of the records were accurate and up to date. Part of their workflow would be to search for the provider (with some indicators such as their address or NPI number) and verify that the information was accurate.

Collaboration

It took many meetings to align our knowledge and expertise of the workflows to design smart solutions. It was amazing to see members of the team from development to data quality collaborate creatively. Facilitating those meetings was a highlight of my experience at InterSystems.

Design

Leading with design is new to the Healthcare. The fancy word is “Pragmatic Interoperability” - designing around workflows. Our goal for the Master Browser was to increase simplicity navigating through providers, clinics, and insurance networks. The previous version had a clickable chart, but it was difficult to keep track of recent searches.

Usability Test

For the usability test provided the script, setup, moderating, and parsed the findings. The new design delivered a smart way through these records with cards. In our testing sessions we learned a few important things: users were able to navigate through the Master Browser easily, but more complicated pages such as History were not as clear to navigate. The information needed to be revisited because it didn't match what the user was expecting.

Future

Provider Directory is still being developed, but the team has embraced the need for good design. We aim to create efficient experiences for the user’s workflow and not just build around the data. A better experience is one of the ways Healthcare UI will continue to be improved and save lives.
